3//! # libmodbuzz
4//!
5//! A safe, idiomatic Rust implementation of the Modbus protocol supporting
6//! RTU (serial) and TCP (Ethernet) transports.
7//!
8//! The crate exposes protocol enums, transport-independent PDUs, RTU and TCP
9//! frame codecs, an in-memory Modbus data model, and ready-to-use RTU/TCP
10//! server implementations.
11//!
12//! ## Quick Start
13//!
14//! ```rust
15//! use libmodbuzz::server::rtu::RtuServer;
16//! use libmodbuzz::server::ModbusServer;
17//!
18//! // Create an RTU server on slave address 1
19//! let mut server = RtuServer::new(1);
20//!
21//! // Initialize some data
22//! server.data.write_coil(0, true);
23//! server.data.write_holding_register(0, 0x1234);
24//!
25//! // Process a Read Coils request (FC 01)
26//! let request = vec![0x01, 0x01, 0x00, 0x00, 0x00, 0x08, 0x3D, 0xCC];
27//! let response = server.process(&request);
28//! assert!(response.is_some());
29//! ```
30//!
31//! ## Architecture
32//!
33//! The library is organized into clean, separated modules:
34//!
35//! - [`protocol`] — Modbus function codes, exception codes, diagnostic codes
36//! - [`error`] — Unified error handling with [`ModbusError`]
37//! - [`checksum`] — CRC-16 (RTU) and LRC (ASCII) calculations
38//! - [`frame`] — Protocol Data Unit and transport-specific frame types
39//! - [`data_model`] — Four-zone data model (coils, inputs, registers, all zero-based)
40//! - [`command`] — Function code dispatch and processing
41//! - [`server`] — Shared server trait with RTU and TCP implementations
42//!
43//! ## Supported Function Codes
44//!
45//! | Code | Name | Status | Notes |
46//! |------|------|--------|-------|
47//! | FC 01 | Read Coils | Implemented | |
48//! | FC 02 | Read Discrete Inputs | Implemented | |
49//! | FC 03 | Read Holding Registers | Implemented | |
50//! | FC 04 | Read Input Registers | Implemented | |
51//! | FC 05 | Write Single Coil | Implemented | |
52//! | FC 06 | Write Single Register | Implemented | |
53//! | FC 07 | Read Exception Status | Defined only | Present in [`FunctionCode`], currently returns `IllegalFunction`. |
54//! | FC 08 | Diagnostics | Partially implemented | Supports a subset of [`DiagnosticsCode`] values. |
55//! | FC 15 | Write Multiple Coils | Implemented | |
56//! | FC 16 | Write Multiple Registers | Implemented | |
57//!
58//! ## Addressing
59//!
60//! Public APIs in this crate use zero-based addresses. If your device map is
61//! documented with Modbus reference numbers such as `00001` or `40001`, convert
62//! those to zero-based offsets before calling library functions.
