1. Overview

Modbus is an industrial communication protocol developed in 1979. It is a client/server architecture. Modbus RTU is the most common variant, typically used over RS-485 serial connections.

sequenceDiagram participant Client participant Server Client->>Server: Request Frame Server-->>Client: Response Frame

2. Data Model

The protocol defines four basic data types:

Data Type Object Type Access Description
Coils Single bit Read/Write Discrete outputs, alters state of an ON/OFF device.
Discrete Inputs Single bit Read-only Discrete inputs, provided by an I/O system.
Input Registers 16-bit word Read-only Analog inputs, provided by an I/O system.
Holding Registers 16-bit word Read/Write Configuration values, analog outputs.

3. Frame Structure (RTU)

The RTU Modbus frame consists of an Application Data Unit (ADU) which encapsulates a Protocol Data Unit (PDU).

4. Function Codes

Function Code Name Description
01 (0x01)Read CoilsRead from 1 to 2000 contiguous status of coils.
02 (0x02)Read Discrete InputsRead from 1 to 2000 contiguous status of discrete inputs.
03 (0x03)Read Holding RegistersRead the contents of a contiguous block of holding registers.
04 (0x04)Read Input RegistersRead from 1 to 125 contiguous input registers.
05 (0x05)Write Single CoilWrite a single output to either ON or OFF.
06 (0x06)Write Single RegisterWrite a single holding register.
08 (0x08)DiagnosticsSeries of tests for checking the communication system.
15 (0x0F)Write Multiple CoilsWrite a sequence of coils to either ON or OFF.
16 (0x10)Write Multiple RegistersWrite a block of contiguous registers.

5. CRC-16 Checksum

The RTU format uses a CRC-16 polynomial 0xA001 for error detection. The CRC is appended to the message low byte first, then high byte.

def calculate_crc16(data: bytes) -> int:
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 1:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return crc

6. Exception Responses

When the server cannot process the request, it replies with an exception response. The function code in the response is the request function code + 0x80.

Code Name Description
01ILLEGAL FUNCTIONThe function code received in the query is not an allowable action for the server.
02ILLEGAL DATA ADDRESSThe data address received in the query is not an allowable address for the server.
03ILLEGAL DATA VALUEA value contained in the query data field is not an allowable value for server.
04SLAVE DEVICE FAILUREAn unrecoverable error occurred while the server was attempting to perform the requested action.

1. Overview

Modbus TCP encapsulates Modbus PDU payloads into TCP packets. It operates over TCP port 502. The protocol replaces the slave address and CRC from the RTU frame with a specific header called MBAP (Modbus Application Protocol).

2. Data Model

The data model is identical to Modbus RTU (Coils, Discrete Inputs, Input Registers, Holding Registers).

3. Frame Structure (TCP/MBAP)

The MBAP header is 7 bytes long and precedes the standard Modbus PDU.

┌──────────────┬────────────┬─────────┬─────────┬──────────────┬──────────────┐
│Transaction ID│Protocol ID │ Length  │ Unit ID │ Function Code│     Data     │
├──────────────┼────────────┼─────────┼─────────┼──────────────┼──────────────┤
│   2 bytes    │  2 bytes   │ 2 bytes │ 1 byte  │    1 byte    │  0-252 bytes │
└──────────────┴────────────┴─────────┴─────────┴──────────────┴──────────────┘
                    MBAP Header                             PDU
Field Size Description
Transaction ID 2 bytes Used for transaction pairing. The server copies the request transaction ID into the response.
Protocol Identifier 2 bytes 0 = Modbus protocol.
Length 2 bytes Number of following bytes (Unit ID + PDU).
Unit Identifier 1 byte Slave address for routing purposes (e.g., Modbus TCP to RTU gateway). Typically 0xFF or 1 for direct IP connections.

4. Function Codes

Function codes remain identical across RTU and TCP variants. Reference the RTU section for a list of common function codes.

5. Exception Responses

Exception encoding is identical to RTU. The server returns the requested function code with the most significant bit set (+ 0x80), followed by a 1-byte exception code.
